# AI Customer Support Assistant

A full-stack AI-powered customer support application that lets users ask questions via text or voice and receive intelligent responses generated by Google Gemini. All conversations are persisted in MongoDB and viewable through an admin logs panel.

## 1. Overview

The AI Customer Support Assistant is a full-stack web application built to automate customer support interactions using Google's Gemini 2.5 Flash language model. Customers can type or speak their queries, receive polished AI-generated responses in real time, and all interactions are automatically stored in MongoDB for admin review.

The backend is a Java Spring Boot REST API. The frontend is a React single-page application with voice input support via the Web Speech API. The two communicate over HTTP, with CORS enabled for local development.

---

## 2. Key Features

- **Text and Voice Input** — customers can type queries or use the built-in microphone button powered by `react-speech-recognition`
- **Gemini AI Responses** — queries are forwarded to the `gemini-2.5-flash` model with a professional customer support system prompt
- **Plain-text Formatting** — raw Markdown output from Gemini is cleaned and normalized before being returned to the client
- **Persistent Chat Logs** — every query and response is saved to MongoDB with a timestamp and input type (`text` or `voice`)
- **Admin Logs Panel** — a dedicated view in the UI to browse all conversation history, with timestamps and input type indicators
- **Log Management** — admins can delete all logs from the UI with a confirmation prompt
- **Markdown Rendering** — AI responses are rendered using `react-markdown` for proper formatting in the chat window
- **Auto-scroll** — the chat window automatically scrolls to the latest message
- **CORS Enabled** — the backend uses `@CrossOrigin` to allow requests from the React development server

## 8. Configuration

### Backend — Gemini API Key

Open `Config.java` and replace the API key with your own:

```java
// src/main/java/.../Config.java
private static final String API_KEY = "YOUR_GEMINI_API_KEY_HERE";
```

> **Note:** For production use, move the API key to `application.properties` or an environment variable instead of hardcoding it.

### Backend — MongoDB Connection

The MongoDB URI is configured in `application.properties`:

```properties
spring.application.name=customer.support.assistant
spring.data.mongodb.uri=mongodb://localhost:27017/ai_support
```

Change the URI if your MongoDB instance runs on a different host, port, or requires authentication:

```properties
spring.data.mongodb.uri=mongodb://username:password@host:port/ai_support
```

### Frontend — Backend URL

The frontend points to `http://localhost:8080` by default (set directly in `Chat.js` and `Logs.js` via axios calls). If your backend runs on a different port, update the base URL in both files.

## 10. API Reference

### POST `/chat`

Sends a customer query to the AI and saves the conversation to MongoDB.

**Request Body:**

```json
{
  "query": "How do I reset my password?",
  "inputType": "text"
}
```

| Field       | Type   | Description                          |
|-------------|--------|--------------------------------------|
| `query`     | String | The customer's question or message   |
| `inputType` | String | `"text"` or `"voice"`                |

**Response:**

```json
{
  "response": "To reset your password, please visit the login page and click..."
}
```

---

### GET `/admin/logs`

Returns all stored conversation logs from MongoDB.

**Response:**

```json
[
  {
    "id": "64f1a2b3c4d5e6f7a8b9c0d1",
    "query": "How do I reset my password?",
    "response": "To reset your password...",
    "inputType": "text",
    "timestamp": "2025-07-05T10:30:00"
  }
]
```

---

### DELETE `/admin/logs`

Deletes all conversation logs from MongoDB.

**Response:**

```
All logs deleted
```

## 11. Frontend Components

### `App.js`

Root component. Manages a single boolean state (`showLogs`) to toggle between the `Chat` view and the `Logs` view. No routing library is used — it's a simple conditional render.

### `Chat.js`

The main customer-facing chat interface. Key behaviors:

- Maintains a local `messages` array for the current session's conversation history
- Sends queries to `POST /chat` using axios
- Supports **voice input** via `react-speech-recognition` — the microphone button starts listening and populates the input field with the transcript
- The microphone icon turns red while listening
- Renders bot responses using `<ReactMarkdown>` for proper formatting
- Pressing **Enter** or clicking the send button submits the query
- The chat window auto-scrolls to the latest message

### `Logs.js`

Admin panel for viewing all past conversations. Key behaviors:

- Fetches all logs from `GET /admin/logs` on mount
- Displays each log entry as a user/bot message pair with timestamp and input type metadata
- The "Delete Logs" button calls `DELETE /admin/logs` with a browser confirmation dialog
- Uses `<ReactMarkdown>` to render stored AI responses consistently

### `Chat.css`

Single stylesheet covering the full UI including the chat container, message bubbles (user vs. bot), input bar, microphone and send buttons, log action buttons, and responsive layout.

## 13. How It Works

1. The customer types or speaks a query in the React chat UI.
2. The frontend sends a `POST /chat` request with the query and input type.
3. The Spring Boot `Controller` receives the request and calls `AiService.generateResponse()`.
4. `AiService` builds a prompt with a professional customer support system instruction prepended to the customer's query, then calls the `gemini-2.5-flash` model via the Google GenAI SDK.
5. The raw response from Gemini is cleaned by `formatToPlainText()` — stripping excess asterisks, normalizing list formatting, and collapsing extra blank lines.
6. The cleaned response is returned to the controller, which saves the full interaction (`query`, `response`, `inputType`, `timestamp`) to MongoDB via the `Repository`.
7. The response is returned to the frontend as `{ "response": "..." }`.
8. The React app appends both the user message and the AI response to the local messages state, rendering them in the chat window.
9. Admins can view or delete all stored logs through the Logs panel.

## 15. Limitations

- **API key is hardcoded** in `Config.java` — not suitable for production without moving it to environment variables
- **No authentication** on admin endpoints (`/admin/logs`) — anyone with network access can view or delete logs
- **No session persistence** — chat history in the UI is in-memory only and lost on page refresh (logs are persisted in MongoDB, but the active session is not reloaded)
- **Voice input** requires a browser that supports the Web Speech API (Chrome recommended; not supported in all browsers)
- **Single-tenant** — no concept of separate users or sessions; all chats share one log collection
- **Dependent on Gemini API availability** — the app requires an active internet connection and a valid API key with sufficient quota
